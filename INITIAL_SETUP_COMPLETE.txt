================================================================================
  QUICK START
================================================================================

1. Clone and Setup:
   $ git clone https://github.com/CKCHDX/SLP.git
   $ cd SLP
   $ python -m venv venv
   $ source venv/bin/activate  # Linux/Mac
   $ pip install -r requirements.txt

2. Create Configuration:
   $ mkdir -p config
   $ cp config/csh.yaml.example config/csh.yaml  # (create from template)

3. Run CSH:
   $ python csh/csh.py

4. Access Web Interfaces:
   - DCC (Control): http://localhost:9000/dcc
   - SLC (Monitoring): http://localhost:9000/slc

5. Run Tests:
   $ pytest
   $ pytest --cov=csh --cov=slp

================================================================================
  DESIGN PRINCIPLES
================================================================================

1. MODULARITY
   - CSH and SLP are separate but integrated
   - Clear interfaces between modules
   - Each module can be tested independently
   - Easy to extend without affecting others

2. ENCRYPTION BY DEFAULT
   - All inter-service communication encrypted
   - Multiple encryption options (TLS, DTLS, Noise)
   - Centralized key management
   - No plaintext allowed

3. WEB-BASED INTERFACES
   - DCC for service control and configuration
   - SLC for real-time monitoring and debugging
   - Localhost-only for security
   - WebSocket for live updates

4. CONFIGURATION AS CODE
   - YAML-based configuration
   - Version control friendly
   - Environment-specific configs
   - No GUI configuration (yet)

5. SCALABILITY
   - Service-based architecture
   - Process isolation
   - Independent scaling
   - Future clustering support
